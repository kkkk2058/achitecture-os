## CH 2. 컴퓨터 구조 시작하기

### 1. 핵심 개념 3개

- 컴퓨터 구조: 컴퓨터가 이해하는 정보(데이터, 명령어)와 부품으로 나뉜다
- CPU: ALU,레지스터,제어장치로 이루어져 있다
- 시스템 버스의 내부 구성:
  - 주소 버스: 주소를 주고받는 통로 (읽기 순서 2)
  - 데이터 버스: 명령어, 데이터 주고받는 통로 (읽기 순서 3)
  - 제어 버스: 제어 신호를 주고받는 통로 (읽기 순서 1)

### 2. 내 말로 설명하기

좋은 개발자가 되려면 단순히 프로그래밍 언어를 잘 하는 것만이 아닌 컴퓨터의 근간을 알아야 한다. 우리는 현실 세계에서의 성능과 용량, 비용을 고려해야하기 때문이다. 컴퓨터가 이해하는 2가지 정보는 데이터와 명령어이다. 부품 4가지도 존재한다. 메모리, CPU, 보조 기억 장치, 입출력 장치가 존재하며 이들은 메인보드로 연결되어 있고 시스템 버스라는 통로를 통해서 정보를 주고받는다. 

### 3. 헷갈려서 AI에게 확인한 부분

- 처음 헷갈렸던 내용: 시스템 버스란 정확히 무엇이지?
- AI에게 물어본 질문: 시스템 버스란 어떤 개념인가?
- AI 답변을 보고 이해한 내용: 데이터를 전송하기 위한 통로이나 물리적인 통로임을 이해했다. 개별적으로 1대1 연결이 아니라 공용 통로를 공유하여 복잡도를 줄이고 유기적으로 정보 교환하기에 시스템 버스라는 용어를 사용했음
- 아직 남은 질문: x

### 4. 개발 또는 일상과 연결해 본 예시

VLM 모델 추론 상황: Qwen2.5-VL 같은 모델을 로컬에서 실행할 때, 모델 파라미터(데이터)는 보조기억장치에서 메모리로 로드된다. CPU(또는 GPU) 내부의 제어장치는 이 데이터를 처리하라는 명령어를 내리고, ALU는 행렬 연산을 수행한다. 이때 메모리 용량이 부족하면 스와핑이 발생하여 성능이 저하된다.(하드웨어 구조적 한계를 소프트웨어가 마주하는 전형적인 사례)

## CH 3. 데이터

### 1. 핵심 개념 3개

- 정보 단위: 비트는 0과 1을 표현하는 가장 작은 정보 단위
- 인코딩: 코드화 하는 과정, 문자를 0과1인 문자 코드로 변환
- 한글 인코딩: 한글의 특징(초성,중성,종)에 따라 euc-kr, utf-8이 존재함

### 2. 내 말로 설명하기

0과 1을 표현하는 가장 작은 정보 단위는 비트이다. 1바이트는 8비트이다. 워드는 cpu가 한 번에 처리할 수 있는 크기 단위이다. 이진법을 활요해서 음수도 표현할 수 있는데 2의 보수를 활용하여 계산한다. 문자 집합은 컴퓨터가 이해할 수 있는 문자들이고 문자를 0과1로 변환하는 인코딩 반대의 과정은 디코딩이다. 한글 인코딩은 한글의 특징을 고려해서 만들어졌다. 현재는 utf-8을 주로 활용한다.

### 3. 헷갈려서 AI에게 확인한 부분

- 처음 헷갈렸던 내용: 2진법의 음수 표현
- AI에게 물어본 질문: 2진법의 음수 표현 하는법이 왜 2의 보수를 활용할까?
- AI 답변을 보고 이해한 내용: 0의 유일성, 뺄셈의 가산화가 가능하다
- 아직 남은 질문: x

### 4. 개발 또는 일상과 연결해 본 예시

모델 경량화와 정보 단위: LLM와 같은 대형 모델을 다룰 때, 가중치를 FP16(16비트)에서 INT8(8비트) 또는 INT4(4비트)로 양자화(Quantization)하는 작업은 결국 정보 단위를 줄여 메모리 사용량을 절감하는 과정이다. 16비트 데이터를 4비트로 줄이면 이론적으로 메모리 사용량이 1/4로 감소하여 로컬 GPU 환경에서 추론이 가능하도록 만들 수 있다.

센서 데이터와 이진법: XR 기기의 IMU 센서나 포인트 클라우드 좌표 데이터는 정밀도에 따라 비트 수가 결정된다. 만약 가속도 값을 8비트로 표현한다면 -128부터 127까지만 표현 가능하므로, 급격한 움직임 발생 시 데이터가 허용 범위를 넘어서는 오버플로우(Overflow)가 발생해 경로 추적에 오류가 생길 수 있다.

멀티모달 로그와 UTF-8: Whisper를 통해 추출한 음성 텍스트나 BLIP으로 생성한 이미지 캡셔닝 결과를 JSON으로 저장할 때, 인코딩을 utf-8로 설정하지 않으면 한글 로그가 깨지는 현상이 발생한다.
