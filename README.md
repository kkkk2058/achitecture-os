### 4. 개발 또는 일상과 연결해 본 예시

모델 경량화와 정보 단위: LLM와 같은 대형 모델을 다룰 때, 가중치를 FP16(16비트)에서 INT8(8비트) 또는 INT4(4비트)로 양자화(Quantization)하는 작업은 결국 정보 단위를 줄여 메모리 사용량을 절감하는 과정이다. 16비트 데이터를 4비트로 줄이면 이론적으로 메모리 사용량이 1/4로 감소하여 로컬 GPU 환경에서 추론이 가능하도록 만들 수 있다.

센서 데이터와 이진법: XR 기기의 IMU 센서나 포인트 클라우드 좌표 데이터는 정밀도에 따라 비트 수가 결정된다. 만약 가속도 값을 8비트로 표현한다면 -128부터 127까지만 표현 가능하므로, 급격한 움직임 발생 시 데이터가 허용 범위를 넘어서는 오버플로우(Overflow)가 발생해 경로 추적에 오류가 생길 수 있다.

멀티모달 로그와 UTF-8: Whisper를 통해 추출한 음성 텍스트나 BLIP으로 생성한 이미지 캡셔닝 결과를 JSON으로 저장할 때, 인코딩을 utf-8로 설정하지 않으면 한글 로그가 깨지는 현상이 발생한다.

### 3. 헷갈려서 AI에게 확인한 부분

- 처음 헷갈렸던 내용: 명령어 주소 지정 방식이 다양하게 있는 이유
- AI에게 물어본 질문: 명령어 주소 지정 방식이 다양하게 있는 이유

- 아직 남은 질문: 실제로 어떤 상황일 때 이 지정방식이 달라질까?

    상황: for(int i=0; i<1000; i++)에서 i의 값 변화.
    상황: int count = 10; 또는 x = x + 1;

  - 컴퓨터는 0과 1로만 이루어져 있다는 것을 전달한다. 이후에 한글을 영어로 번역하는 것처럼 문자를 번역하는 과정임을 이해시킨 후, 평소에 음식을 만들 때 재료를 냉장고에서 꺼내서 하지 마트에 들리거나 실제로 바다에 가서 해산물을 매번 잡는 것은 어렵고 시간도 오래걸리기 때문에 미리 사용할 재료를 냉장고에 넣어둔 다음 활용한다는 방식으로 설명한다.

### 2. 내 말로 설명하기
